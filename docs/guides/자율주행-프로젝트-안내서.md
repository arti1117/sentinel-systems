# 자율주행 프로젝트 워크스페이스 안내서

> 이 문서는 프로그래밍이나 로봇공학 배경 없이도 이 저장소의 구조와 목적을 이해할 수 있도록 작성되었습니다.

---

## 1. 이 저장소는 뭘 하는 곳인가요?

한마디로, **자율주행 자동차를 만들기 위한 소프트웨어 모음집**입니다.

자율주행차는 크게 세 가지가 필요합니다:

1. **두뇌** — 주변을 인식하고, 판단하고, 차를 조종하는 소프트웨어
2. **연습장** — 실제 도로에 나가기 전에 컴퓨터 안에서 시뮬레이션하는 가상 환경
3. **축소 모형** — 작은 자동차로 먼저 실험해보는 교육/연구용 플랫폼

이 저장소는 이 세 가지를 각각 담당하는 세 개의 폴더로 나뉩니다:

| 폴더 | 역할 | 비유 |
|------|------|------|
| `AutowareFoundation/` | 자율주행 두뇌 소프트웨어 | 실제 차에 들어가는 "운전사 AI" |
| `CarlaSimulator/` | 가상 세계 시뮬레이터 | 운전 연습을 하는 "가상 도시" |
| `F1Tenth/` | 1/10 크기 소형 레이싱카 | 대학교 수업/대회용 "RC카 실험실" |

## 2. AutowareFoundation — 자율주행의 두뇌

### 이게 뭔가요?

Autoware는 **실제 자율주행차에 탑재되는 소프트웨어**입니다. 사람이 운전할 때 하는 모든 것을 소프트웨어로 구현한 것이죠.

### 사람의 운전 과정과 비교

사람이 운전할 때를 떠올려 보세요:

```
눈으로 본다 → 어디 있는지 파악 → 주변 상황 인식 → 어디로 갈지 계획 → 핸들/브레이크 조작 → 차가 움직임
```

Autoware도 똑같은 순서로 동작합니다:

```
센서로 감지(Sensing) → 위치 파악(Localization) → 주변 인식(Perception) → 경로 계획(Planning) → 조종(Control) → 차량 제어(Vehicle)
```

각 단계를 담당하는 소프트웨어 조각을 **"패키지"**라고 부르며, 이것들이 서로 데이터를 주고받으며 협업합니다.

### 주요 저장소 역할

| 저장소 | 역할 | 비유 |
|--------|------|------|
| `autoware` | 전체를 묶는 설계도 | 레고 조립 설명서 |
| `autoware_core` | 검증된 핵심 부품 (69개 패키지) | 정식 출시된 부품 |
| `autoware_universe` | 실험 중인 확장 부품 | 베타 테스트 중인 부품 |
| `autoware_msgs` | 부품 간 통신 규격 | 부품끼리 대화하는 "공용어" |
| `autoware_utils` | 여러 부품이 공유하는 도구 모음 | 공용 공구함 (12개 분야) |
| `autoware_launch` | 전체를 한 번에 켜는 실행 설정 | 자동차 시동 버튼 |
| `autoware_tools` | 디버깅/보정/성능 분석 도구 | 정비소 장비 |

### "ROS 2"란?

모든 Autoware 코드는 **ROS 2(Robot Operating System 2)**라는 프레임워크 위에서 돌아갑니다. ROS 2는 로봇 소프트웨어의 "운영체제" 같은 것입니다.

- 각 기능(센서 처리, 경로 계획 등)은 독립적인 프로그램(**노드**)으로 돌아감
- 노드들은 **"토픽"**이라는 게시판에 데이터를 올리고 읽으며 서로 소통함
- 예: 카메라 노드가 "앞에 사람 있음"을 올리면 → 계획 노드가 읽고 "멈춰야겠다" 판단

### 빌드 (코드를 실행 가능한 프로그램으로 만드는 과정)

```bash
# 1단계: ROS 2 환경 불러오기 (도구 준비)
source /opt/ros/humble/setup.bash

# 2단계: 전체 빌드 (모든 부품 조립)
colcon build --parallel-workers $(nproc --all) --executor parallel

# 특정 패키지 하나만 빌드하고 싶을 때
colcon build --packages-select 패키지이름

# 테스트 실행 (제대로 작동하는지 검증)
colcon test --packages-select 패키지이름
colcon test-result --verbose
```

> **colcon**은 ROS 2의 "빌드 도구"입니다. 소스 코드를 실행 가능한 프로그램으로 변환해 줍니다.

## 3. CarlaSimulator — 가상 도시에서 연습하기

### 이게 뭔가요?

CARLA는 **컴퓨터 안에 만든 가상 도시**입니다. 게임 엔진(Unreal Engine)으로 만들어져서 실제처럼 보이는 도로, 건물, 차량, 보행자가 있습니다.

왜 필요할까요? 자율주행차를 실제 도로에서 바로 테스트하면 위험하고 비용이 많이 듭니다. 가상 세계에서 수천 번 연습한 뒤 실제로 나가는 것이 안전합니다.

### 핵심 구성요소

| 저장소 | 역할 | 비유 |
|--------|------|------|
| `carla` | 가상 도시 자체 | 게임 맵 (도로, 건물, 날씨) |
| `scenario_runner` | 상황극 연출 | "갑자기 보행자가 뛰어나오는" 시나리오 설정 |
| `ros-bridge` | Autoware와 연결하는 다리 | 가상 세계의 센서 데이터를 Autoware 형식으로 번역 |
| `leaderboard` | 자율주행 AI 성적표 | 얼마나 잘 운전하는지 점수 매기는 평가 시스템 |

### 보조 도구들

| 저장소 | 역할 | 비유 |
|--------|------|------|
| `map` | 도로 지도 데이터 처리 라이브러리 | 네비게이션의 지도 엔진 |
| `Scenic` | 시나리오를 코드로 작성하는 전용 언어 | "비 오는 날 교차로에서 좌회전" 같은 상황을 프로그래밍 |
| `sumo` | 대규모 교통 흐름 시뮬레이션 | 도시 전체의 차량 흐름을 재현 |
| `RobotecGPULidar` | GPU 가속 라이다 센서 시뮬레이션 | 가상 레이저 스캐너 |
| `DReyeVR` | VR 헤드셋으로 가상 차 안에 앉기 | 1인칭 시점 VR 운전 체험 |
| `carla-digitaltwins` | 실제 지도(OpenStreetMap)를 가상 도시로 변환 | 구글맵 → 게임 맵 변환기 |
| `rllib-integration` | 강화학습으로 AI 운전 훈련 | AI에게 "잘하면 상, 못하면 벌"로 운전 가르치기 |
| `carla-autoware` | CARLA 안에서 Autoware 두뇌 테스트 | 가상 차에 실제 AI 두뇌 장착 |

### 주요 용어

- **Unreal Engine (UE)**: 게임을 만드는 엔진. CARLA는 이 위에 만들어짐
- **OpenDRIVE**: 도로를 기술하는 국제 표준 형식 (차선, 교차로, 신호등 등)
- **OpenSCENARIO**: 시나리오를 기술하는 국제 표준 형식 (어떤 차가 언제 어디로 가는지)

### 빌드

```bash
# CARLA 본체 빌드 (C++ 프로젝트, 시간이 오래 걸림)
cmake -G Ninja -S . -B Build --toolchain=$PWD/CMake/Toolchain.cmake -DCMAKE_BUILD_TYPE=Release
cmake --build Build

# Python API 설치 (CARLA를 파이썬으로 제어하기 위해)
cmake --build Build --target carla-python-api-install

# ScenarioRunner (파이썬만 필요)
pip install -r requirements.txt
python scenario_runner.py
```

### 버전 호환성 주의사항

CARLA 생태계에서 가장 중요한 것은 **버전 맞추기**입니다. 서로 다른 도구가 서로 다른 CARLA 버전만 지원합니다:

| 도구 | 필요한 CARLA 버전 | 필요한 엔진 |
|------|-------------------|-------------|
| DReyeVR (VR) | 0.9.13 (고정) | UE 4.26 |
| carla-autoware | 0.9.10.1 (고정) | - |
| rllib-integration | 0.9.11 (권장) | - |
| Scenic | 0.9.12 이상 | - |
| carla 본체 (최신) | 0.10.0 | UE 5.5 |

> 잘못된 버전을 섞으면 작동하지 않으므로, 어떤 도구를 쓸지 먼저 정하고 그에 맞는 CARLA 버전을 설치해야 합니다.

## 4. F1Tenth — 소형 레이싱카 실험실

### 이게 뭔가요?

F1Tenth는 **실제 자동차의 1/10 크기인 RC카**에 센서와 컴퓨터를 달아서 자율주행을 연구하는 프로젝트입니다. 대학교 수업과 국제 대회에서 사용됩니다.

핵심 아이디어: **시뮬레이션에서 짠 코드가 실제 RC카에서도 그대로 동작**합니다. 이것이 가능한 이유는 시뮬레이션과 실제 차가 **같은 형식의 데이터**를 주고받기 때문입니다.

### 시뮬레이션과 실제 차의 구조

```
[시뮬레이션 경로]
f1tenth_gym (가상 물리엔진) → f1tenth_gym_ros (ROS 2 변환) → 표준 토픽

[실제 차량 경로]
실제 센서 + VESC 모터 드라이버 → f1tenth_system → 표준 토픽

두 경로 모두 같은 "표준 토픽"을 사용하므로,
내가 만든 자율주행 코드는 수정 없이 양쪽에서 작동합니다.
```

**표준 토픽**(데이터 채널)은 세 가지입니다:

| 토픽 이름 | 데이터 종류 | 비유 |
|-----------|-------------|------|
| `/scan` | 라이다 스캔 데이터 | "주변에 뭐가 있는지" 360도 거리 측정 |
| `/drive` | 조향/속도 명령 | "핸들 왼쪽 30도, 속도 2m/s" |
| `/odom` | 위치/속도 추정값 | "지금 여기에 있고 이 속도로 가고 있음" |

### 주요 저장소 역할

| 저장소 | 역할 | 비유 |
|--------|------|------|
| `f1tenth_gym` | 2D 물리 시뮬레이터 (ROS 없이 단독 실행 가능) | 가상 레이싱 트랙 |
| `f1tenth_gym_ros` | 시뮬레이터를 ROS 2에 연결 | 가상 트랙에 표준 통신 장치 부착 |
| `f1tenth_system` | 실제 RC카의 센서/모터 제어 | RC카의 신경계 |
| `vesc` | 모터 컨트롤러 드라이버 | 모터에 "이 속도로 돌아"라고 전달하는 역할 |
| `f1tenth_racetracks` | 트랙 지도 데이터 | 각 서킷의 지도 파일 |
| `f1tenth_planning` | 알고리즘 라이브러리 (ROS 불필요) | "어떻게 달릴까" 전략 모음 |
| `range_libc` | 고속 레이저 시뮬레이션 C++ 라이브러리 | 가상 라이다의 핵심 엔진 |
| `particle_filter` | 위치 추정 알고리즘 (range_libc 필요) | "지금 내가 트랙 어디에 있지?" 계산기 |
| `f1tenth_labs_openrepo` | 수업 실습 과제 (Lab 1~9, ROS 2) | 교과서 실습 문제집 |
| `teleop_tools` | 조이스틱/키보드로 수동 조종 | RC카 리모컨 소프트웨어 |

### 빌드 순서가 중요한 경우

**위치 추정 기능을 쓰려면:**

```bash
# 1단계: range_libc 먼저 설치 (C++ 라이브러리)
cd F1Tenth/range_libc/pywrapper
python setup.py install
# GPU가 있다면: WITH_CUDA=ON python setup.py install

# 2단계: 그다음 particle_filter 빌드 (range_libc에 의존)
colcon build --packages-select particle_filter
```

> range_libc가 없으면 particle_filter가 작동하지 않습니다. 반드시 순서를 지켜야 합니다.

### 더 이상 사용하지 않는 저장소

| 폐기된 저장소 | 대체 저장소 | 이유 |
|---------------|-------------|------|
| `f1tenth_labs` | `f1tenth_labs_openrepo` | ROS 1 → ROS 2로 이전 |
| `f1tenth_simulator` | `f1tenth_gym_ros` | 더 나은 시뮬레이터로 교체 |

## 5. 자주 등장하는 용어 사전

### 개발 도구

| 용어 | 설명 |
|------|------|
| **ROS 2** | 로봇 소프트웨어를 만드는 프레임워크. 여러 프로그램이 데이터를 주고받으며 협업하는 구조를 제공 |
| **colcon** | ROS 2에서 소스 코드를 실행 가능한 프로그램으로 변환(빌드)하는 도구 |
| **CMake** | C++ 프로젝트의 빌드 설정을 관리하는 도구. "이 파일들을 이렇게 컴파일해라" 라는 레시피 |
| **pip** | Python 패키지를 설치하는 도구 (`pip install 패키지이름`) |
| **npm** | JavaScript/TypeScript 패키지를 설치하는 도구 (`npm install`) |
| **Docker** | 소프트웨어를 "컨테이너"에 담아서 어떤 컴퓨터에서든 동일하게 실행할 수 있게 하는 도구 |
| **Git** | 코드의 변경 이력을 관리하는 버전 관리 도구 |

### 센서와 하드웨어

| 용어 | 설명 |
|------|------|
| **LiDAR (라이다)** | 레이저를 360도로 쏘아서 주변 물체까지의 거리를 측정하는 센서. 자율주행차의 "눈" |
| **VESC** | 소형 전동 모터의 속도와 방향을 정밀하게 제어하는 전자 장치 |
| **GPU** | 그래픽 처리 장치. 시뮬레이션, AI 학습, 센서 처리에서 대량의 계산을 빠르게 수행 |
| **CUDA** | NVIDIA GPU에서 범용 계산을 실행하는 기술 |

### 자율주행 개념

| 용어 | 설명 |
|------|------|
| **Localization (위치 추정)** | "내가 지금 지도상 어디에 있는가?"를 계산하는 것 |
| **Perception (인지)** | 센서 데이터로 "주변에 뭐가 있는가?"를 파악하는 것 |
| **Planning (경로 계획)** | "어디로 어떻게 갈 것인가?"를 결정하는 것 |
| **Control (제어)** | 계획대로 움직이기 위해 핸들과 페달을 실제로 조작하는 것 |
| **SLAM** | 지도를 만들면서 동시에 자기 위치를 파악하는 기술 |
| **Particle Filter** | 수천 개의 "추측 위치"를 뿌린 뒤, 센서 데이터와 맞는 것만 살려서 위치를 추정하는 알고리즘 |
| **Pure Pursuit** | 전방의 목표점을 향해 곡선으로 따라가는 단순하고 효과적인 조향 알고리즘 |
| **MPC (Model Predictive Control)** | 미래 몇 초를 예측하면서 최적의 조작을 계산하는 고급 제어 기법 |

### 데이터 형식

| 용어 | 설명 |
|------|------|
| **Topic (토픽)** | ROS에서 프로그램 간에 데이터를 주고받는 채널. 게시판처럼 올리고(publish) 읽는(subscribe) 구조 |
| **AckermannDriveStamped** | "핸들 각도 + 속도" 정보를 담은 표준 메시지 형식 |
| **LaserScan** | 라이다 센서가 보내는 "거리 측정값 배열" 메시지 형식 |
| **Lanelet2** | 자율주행용 정밀 지도 형식. 차선, 교차로, 신호등, 규제 정보 포함 |
| **OpenDRIVE** | 도로의 기하학적 구조를 기술하는 국제 표준 XML 형식 |
| **Point Cloud** | 라이다가 측정한 3D 점들의 집합. "점들의 구름"으로 주변 형태를 표현 |

```
│  ┌──────────────┐     ros-bridge      ┌──────────────┐  │
│  │   Autoware   │◄──────────────────►│    CARLA      │  │
│  │  (자율주행    │  carla-autoware     │  (가상 도시   │  │
│  │   두뇌)      │  carla-agent-ctrl   │   시뮬레이터) │  │
│  └──────────────┘                     └──────────────┘  │
│         │                                               │
│         │ 같은 자율주행 개념을 소규모로 실험                 │
│         ▼                                               │
│  ┌──────────────┐                                       │
│  │   F1Tenth    │                                       │
│  │  (소형 RC카  │                                       │
│  │   실험실)    │                                       │
│  └──────────────┘                                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

- **Autoware**는 실제 자율주행차에 탑재되는 소프트웨어입니다
- **CARLA**는 그 소프트웨어를 안전하게 테스트하는 가상 환경입니다
- **F1Tenth**는 같은 개념을 작은 규모로 배우고 실험하는 교육 플랫폼입니다
- 세 프로젝트는 ROS라는 공통 통신 프레임워크를 사용하여 서로 연결될 수 있습니다

## 7. 실행 환경 요약

| 프로젝트 | 운영체제 | 주요 언어 | 필수 도구 |
